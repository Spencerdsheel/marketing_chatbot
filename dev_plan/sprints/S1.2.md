## Goal
Authenticate an admin/agent by email + password and issue an HS256 JWT in an httpOnly cookie. No token in the
response body. This is the entry point for the whole admin/agent session model; verification of that cookie
(the auth dependency + `/auth/me`) is the next sprint (S1.3).

## Decisions (locked)
- **JWT lib:** `pyjwt`. **HS256**, signed with `settings.jwt_secret`. Payload:
  `{sub, role, tenant_id, project_ids, iat, exp, jti}` (role as its string value; `tenant_id` may be null for
  PLATFORM_ADMIN). `jti = uuid4().hex` (used by the S1.4 logout blacklist). Default TTL 3600s.
- **Service-specific settings:** add `services/api/src/api/config.py` →
  `class ApiSettings(common.settings.Settings)` extending the shared settings with:
  `cookie_secure: bool = True`, `cookie_samesite: Literal["lax","strict","none"] = "lax"`,
  `cookie_name: str = "access_token"`, `access_token_ttl_seconds: int = 3600`; plus a cached `get_api_settings()`.
  Switch `app.py` to use `get_api_settings()` (it's a drop-in superset of `Settings`). This keeps `services/common`
  untouched while giving the api its own config seam.
  - **Dev note:** over `http://localhost`, a `Secure` cookie won't round-trip in browsers; the `.env` should set
    `COOKIE_SECURE=false` for local testing. Default stays `True` (prod-safe).
- **Login is the one legitimate pre-auth query:** `get_user_by_email(db, email)` is intentionally UNSCOPED
  (there are no claims yet) — it resolves identity. It looks up by `lower(email)=lower($1)`, returns the row
  incl. `password_hash`. The route NEVER returns `password_hash`. `tenant_id` comes from the user row, never
  from the request.
- **No user enumeration:** unknown email, wrong password, and inactive user all return the same `401
  UNAUTHENTICATED`. Password check via `common.crypto.verify_password` (constant-time).
- **Deferred (not this sprint):** rate limiting (→ S2.2), persistent audit-trail table (→ S11.1). Log
  login success/failure via `common.logging` (event fields only, no secrets/PII beyond user id) for now.

## Scope (build) — services/api
1. `pyproject.toml`: add `pyjwt` to dependencies.
2. `api/config.py`: `ApiSettings` + `get_api_settings()` as above; update `app.py` to use it.
3. `api/auth/tokens.py`:
   - `create_access_token(claims: AuthClaims, *, secret: str, ttl_seconds: int) -> tuple[str, str]` →
     `(token, jti)`. Encodes the payload above with `iat`/`exp` (UTC).
   - `decode_access_token(token: str, *, secret: str) -> dict` → validated payload; raises
     `AuthenticationError` (`UNAUTHENTICATED`) on expired/invalid/bad-signature. (Used fully in S1.3; included
     now with tests.)
   - `claims_from_payload(payload: dict) -> AuthClaims` → reconstruct `AuthClaims` (validates the invariant).
4. `api/auth/repository.py` (or `service.py`): `async def get_user_by_email(db, email) -> Row | None` —
   parameterized, case-insensitive, unscoped (documented why).
5. `api/auth/routes.py`:
   - `POST /auth/login`, body `LoginRequest{ email: str, password: str }` (Pydantic; 422 on malformed body).
   - Flow: look up user → if missing/inactive/bad password → `AuthenticationError`. Else build `AuthClaims`
     from the row, mint the token, `response.set_cookie(cookie_name, token, httponly=True,
     secure=settings.cookie_secure, samesite=settings.cookie_samesite, max_age=ttl, path="/")`, return profile
     `{id, email, role, tenant_id, name}` (NO token, NO password_hash).
   - Update `last_login_at` (best-effort).
   - Wire the router into `create_app()`.

## Tests (TDD, services/api/tests/unit)
- `tokens`: `create_access_token` round-trips through `decode_access_token`; payload has sub/role/tenant_id/jti
  and a future `exp`; tampered token, wrong secret, and expired token each raise `AuthenticationError`;
  `claims_from_payload` rebuilds the right `AuthClaims` (incl. PLATFORM_ADMIN null tenant).
- `login` (httpx `ASGITransport`, DB stubd via `app.state` so no real DB): hash a known password with
  `common.crypto.hash_password` for the stub user row.
  - success → 200, `Set-Cookie` present with `HttpOnly` + correct `SameSite`; body has profile, NO token, NO
    password_hash; **RBAC/tenancy**: a CLIENT_ADMIN row yields a token whose claims carry that tenant_id, a
    PLATFORM_ADMIN row yields null tenant_id.
  - wrong password → 401 `UNAUTHENTICATED`, no cookie.
  - unknown email → 401 (identical shape — no enumeration).
  - inactive user → 401.
