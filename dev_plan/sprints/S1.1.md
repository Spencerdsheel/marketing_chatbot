## Goal
Establish the first real data model — `tenants` and `users` — as a migration, a tenant-scoped
`TenantRepository` built on `services/common` primitives, an idempotent seed (one platform admin + one initial
tenant + one client admin), and the mandatory multi-tenant isolation tests. Expose a temporary debug read
endpoint so the data is visible in Postman before real auth lands (S1.2/S1.3).

## Decisions (locked)
- **No `roles` table.** The new model has four FIXED roles (`common.auth.Role`), enforced in code, carried in
  JWT claims — not per-tenant permission rows (that was the *old* phase-1 model). `role` is a constrained
  column on `users`. (Deviates from the backlog's literal "roles" wording; the skill + CLAUDE.md 4-role model
  win.)
- **IDs are app-generated** `uuid4().hex` (TEXT), matching `common`'s repositories — columns are `text`, no DB
  uuid default.
- **`users.email` is globally unique, case-insensitive** (`unique index on lower(email)`), so admin/agent
  login by email alone can resolve the tenant from the user row (per the auth skill).
- **`users` holds only admin/agent roles** (`PLATFORM_ADMIN`/`CLIENT_ADMIN`/`CLIENT_AGENT`). Visitors are never
  user rows (they're anonymous signed sessions).
- **DB-level tenant invariant:** a CHECK constraint mirrors `AuthClaims` — `PLATFORM_ADMIN` ⇒ `tenant_id IS
  NULL`; the other roles ⇒ `tenant_id IS NOT NULL`.
- The `tenants` table has **no `tenant_id` column**; it is scoped on its own `id` via
  `tenant_filter(claims, column="id")`. So it gets a small **custom repo**, not the generic
  `PostgresRepository`.

## Scope (build)
### 1. Migration `services/api/migrations/versions/0002_tenants_users.py` (down_revision `"0001"`)
- `tenants`: `id text pk`, `name text not null`, `slug text not null unique`,
  `enabled boolean not null default true`, `created_at timestamptz not null default now()`,
  `updated_at timestamptz not null default now()`.
- `users`: `id text pk`, `tenant_id text references tenants(id) on delete cascade`, `email text not null`,
  `role text not null check (role in ('PLATFORM_ADMIN','CLIENT_ADMIN','CLIENT_AGENT'))`,
  `password_hash text not null`, `name text`, `active boolean not null default true`,
  `last_login_at timestamptz`, `created_at/updated_at timestamptz not null default now()`,
  plus CHECK `users_tenant_role_chk` (the invariant above).
- Indexes: `create unique index users_email_lower_uniq on users (lower(email))`,
  `index on users(tenant_id)`, `index on users(role)`.
- `downgrade()` drops both tables (users first).
- Use raw SQL via `op.execute(...)` (matches the no-ORM / hand-written-SQL decision in S0.3).

### 2. `TenantRepository` — `services/api/src/api/tenants/repository.py`
Built directly on `common.db.Database` + `common.tenancy`. Methods (async):
- `get(claims, id) -> Row | None` — `SELECT * FROM tenants WHERE id = $1 {tenant_filter(column="id", next_param=2)}`.
- `list(claims) -> list[Row]` — `SELECT * FROM tenants WHERE TRUE {tenant_filter(column="id")}`.
- `create(claims, data) -> Row` — `require_role(claims, Role.PLATFORM_ADMIN)`; generate `id = uuid4().hex`;
  INSERT `name, slug, enabled` (default true). Return the row.
All SQL parameterized (`$N`); never f-string user values (table/columns are constants — fine).

### 3. Seed — `services/api/src/api/seed.py` (runnable: `python -m api.seed`)
- Connects via `DATABASE_URL_DIRECT` (fallback `DATABASE_URL`) using asyncpg directly (bootstrap, pre-repo).
- Inserts **idempotently** (`ON CONFLICT DO NOTHING` on slug / on `lower(email)`):
  - initial tenant: `slug="initial"`, `name="Initial Tenant"`.
  - platform admin user: email from `SEED_PLATFORM_ADMIN_EMAIL` (default `platform@chatbot.local`),
    `role=PLATFORM_ADMIN`, `tenant_id=NULL`.
  - client admin user: email from `SEED_CLIENT_ADMIN_EMAIL` (default `admin@initial.local`),
    `role=CLIENT_ADMIN`, `tenant_id=<initial tenant id>`.
- **Passwords come from env, never hardcoded:** read `SEED_PLATFORM_ADMIN_PASSWORD` /
  `SEED_CLIENT_ADMIN_PASSWORD`; if unset, generate a random one with `secrets.token_urlsafe(16)` and **print
  it once to stdout** so the user can log in at S1.2. Hash with `common.crypto.hash_password` before insert.
- Print a short summary (emails + tenant id; passwords only if generated).

### 4. Temporary debug endpoint — `services/api/src/api/tenants/routes.py`
- `GET /debug/tenants` and `GET /debug/tenants/{id}` → build `TenantRepository(request.app.state.db)` and call
  `list`/`get`. **TEMP:** until S1.3, construct `AuthClaims(subject="debug", role=Role.PLATFORM_ADMIN,
  tenant_id=None)` in-handler. Add a clear `# TEMP(S1.3): replace with real auth dependency` comment.
