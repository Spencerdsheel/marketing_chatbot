## Goal
Turn the login cookie into enforced identity: a FastAPI dependency that reads the cookie, validates the JWT,
and yields `AuthClaims`. Add `GET /auth/me`, and wire the dependency into `/debug/tenants` so it finally runs
with the REAL caller's claims — making per-role tenant isolation observable end-to-end (PLATFORM_ADMIN sees
all tenants; CLIENT_ADMIN sees only its own).

## Decisions (locked)
- New `services/api/src/api/auth/dependencies.py`:
  - `async def get_current_claims(request: Request) -> AuthClaims` — read the cookie named
    `get_api_settings().cookie_name` from `request.cookies`; if absent → `AuthenticationError` (401). Decode via
    `decode_access_token(token, secret=settings.jwt_secret)` (raises `AuthenticationError` on invalid/expired),
    then `claims_from_payload(payload)`. Return the claims.
  - `def require_roles(*roles: Role)` — a dependency FACTORY returning a dependency that calls
    `get_current_claims` then `common.tenancy.require_role(claims, *roles)` (403 on mismatch). Provided for
    downstream routers; used lightly here.
- `GET /auth/me` (in `api/auth/routes.py`): `claims = Depends(get_current_claims)` → returns
  `{subject, role, tenant_id, project_ids}`.
- `api/tenants/routes.py`: REPLACE the `TEMP(S1.3)` hardcoded PLATFORM_ADMIN claims with
  `claims = Depends(get_current_claims)`; pass the real claims to `TenantRepository.get/list`. Remove the TEMP
  comment. `/debug/tenants` now requires authentication and is tenant-scoped by the caller.
- **Blacklist check is deferred to S1.4** (logout). `get_current_claims` will gain the Redis jti-blacklist
  check there; note it with a `# S1.4: add blacklist check` marker.
